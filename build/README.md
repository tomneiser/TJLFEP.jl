# build/ — run, verify, benchmark, and sysimage scripts

Slurm batch scripts and Julia drivers for running TJLFEP on Perlmutter. All
scripts assume the Julia environment from the repo root (see top-level
`README.md`). Most scripts default `TJLFEP_ROOT` to this checkout and the case to
`examples/DIIID_202017C42_500ms_v3.1`; override via environment variables.

**Submit jobs from `build/` (not from the subdirectories):**

```bash
cd build
sbatch verify/batch_debug_nb6_julia.sh
```

Each batch script `cd`s to `build/` and invokes its Julia driver by sub-path
(e.g. `verify/debug_compare_nb6.jl`), so Slurm `-o/-e` logs and all run outputs
land in `build/` — which is also where `timing/collect_scan20_timing.jl` looks
for `TIMING_RESULT` logs. Run output directories (`fortran_runs/`, `*_tasks/`,
`fileInput*/`, `timing_runs/`, `debug_out_*/`, `*.so`) are regenerated by these
jobs and are gitignored.

## Layout

| Subdir | Contents |
|--------|----------|
| `common/` | Shared GPU/MPS driver (`run_gacode_scan20_mps_task.jl`), MPS launch wrapper (`mps-scan-wrapper.sh`), result merge (`merge_gacode_scan20_array.jl`), sysimage helper (`julia_sysimage.inc.sh`). |
| `verify/` | Fortran-vs-Julia verification (batch runners, distributed drivers, overlay plotters). |
| `run/` | Smoke test, file-based validation, 5-node production scan, merge, self-contained submit template. |
| `timing/` | Timing-vs-N_BASIS sweep (per-backend batch runners, driver, collector, plotter). |
| `sysimage/` | CPU + generic GPU sysimage builders and their precompile workloads. |

The scripts cover five capabilities. The sixth — the FUSE-native IMAS `dd` path —
lives in `src/` and is demonstrated in `examples/ITER/ITERstructExample.jl`.

## 1. Verification against Fortran TGLF-EP (`verify/`)

Runs the same DIII-D case through the canonical Fortran `TGLFEP_driver` (shared
m3739 build at `/global/cfs/cdirs/m3739/gacode_add_d3d/TGLF-EP`, override with
`TGLFEP_DIR`) and through Julia, then overlays the results. Set `TGLFEP_DEBUG=1`
(Fortran) / `TJLFEP_DEBUG=1` (Julia) for matched `[*_DBG]` debug lines.

| Step | Single radius (N_BASIS=6, SCAN_N=1, 1 node) | SCAN_N=20 (10 nodes) |
|------|---------------------------------------------|----------------------|
| Fortran | `sbatch verify/batch_debug_nb6_fortran.sh` | `sbatch verify/batch_debug_nb6_fortran_scan20_10n.sh` |
| Julia | `sbatch verify/batch_debug_nb6_julia.sh` | `sbatch verify/batch_debug_nb6_julia_scan20_10n.sh` |
| Compare | `FORTRAN_DIR=... JULIA_DIR=... verify/compare_debug_nb6.sh` | `FORTRAN_DIR=... JULIA_DIR=... FILE_DIR=... verify/compare_debug_nb6_scan20.sh` |

Drivers: `debug_compare_nb6.jl` (single radius), `debug_compare_nb6_scan20_distributed.jl`
(20-radius `pmap`). Plotters: `plot_alpha_compare.jl` (single-radius α overlay),
`plot_nb6_scan20_compare.jl` (scan overlay). See `docs/REPRODUCE_FORTRAN_MATCH.md`.

## 2. Run + validate on input.gacode / input.TGLFEP (`run/`)

No sysimage required; good first smoke test of a fresh checkout.

```bash
# Quick GPU smoke test (1 node, ~1 radius)
sbatch run/batch_smoke_test.sh           # -> run/smoke_test.jl
# Full single-node file-based validation (CPU or GPU auto-detected)
sbatch run/batch_validate_nosysimage.sh  # -> run/validate_nosysimage.jl
```

## 3. Database generation (GPU, MPS, 1 radius/worker) (`run/` + `common/`)

The validated production layout: 20 radii on 5 GPU nodes (4 radii/node, 1 A100 +
an MPS worker team per radius). `common/run_gacode_scan20_mps_task.jl` is the
Julia analogue of the Fortran `TGLFEP_driver`; `common/mps-scan-wrapper.sh`
starts per-node MPS and pins GPUs; `common/merge_gacode_scan20_array.jl` merges
per-radius outputs into the final `sfmin_scan.txt` / `alpha_*_crit.input`.

```bash
sbatch run/batch_run_scan20_5N.sh
# Fully-documented, self-contained template (edit CONFIG paths at top):
sbatch run/submit_tjlfep_gpu_5N_example.sh
```

## 4. Timing vs N_BASIS (`timing/`)

Sweeps `N_BASIS` (`NB` env) over Fortran CPU, Julia CPU, and Julia GPU and
collects `TIMING_RESULT` markers into a CSV + plot.

```bash
./timing/submit_timing_vs_nbasis.sh           # submits the GPU + CPU sweep (from build/)
# after jobs finish:
julia --project=. timing/collect_scan20_timing.jl   # -> timing_runs/scan20_timing.csv
./timing/plot_scan20_timing.sh                       # -> timing_runs/scan20_timing_*.png
```

Per-backend jobs: `timing/batch_time_scan20_fortran.sh`, `timing/batch_time_scan20_julia_cpu.sh`,
`timing/batch_time_scan20_julia_gpu.sh` (drivers `time_scan20_julia_cpu.jl` for CPU,
reuses the capability-3 MPS driver for GPU).

## 5. Sysimage build + run (`sysimage/`)

Precompiled sysimages remove JIT cost (~110 s/team) for production runs. The
canonical GPU image is `TJLFEP_gpu_generic_sysimage.so` (node-count agnostic);
the CPU image is `TJLFEP_cpu_sysimage.so`. Both are written to `build/`. Keep the
`.so` on a non-purged path (CFS, not `$PSCRATCH`) for reuse across jobs.

```bash
sbatch sysimage/batch_build_gpu_sysimage_generic.sh   # -> sysimage/build_gpu_sysimage_generic.jl (+ precompile_gpu_workload_generic.jl)
sbatch sysimage/batch_build_cpu_sysimage.sh           # -> sysimage/build_cpu_sysimage.jl (+ precompile_cpu_workload.jl)
```

Batch scripts auto-detect the image via `TJLFEP_GPU_SYSIMAGE` / `TJLFEP_SYSIMAGE`
(falling back to JIT if missing). `common/julia_sysimage.inc.sh` is the shared
helper that exports `JULIA_SYSIMAGE_ARGS`.
